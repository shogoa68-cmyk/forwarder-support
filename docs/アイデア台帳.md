## 🟠 中：あると便利・品質向上

- [ ] **社内メモ行の挿入（出力に出ない内部コメント行）**
  - 背景: 既存のリマーク行はプレビュー・PDF・CSV・Excelに出力される。社内向けのメモを行として挿入したいが客先資料には出したくない
  - 設計方針（合意済み）:
    - 新しい行タイプ `_type: 'internal'` を追加（リマーク行のほぼコピー）
    - スタイル: 薄いオレンジ背景＋🔒アイコンでリマーク行と視覚的に区別
    - 全出力（プレビュー・CSV・Excel・PDF）から除外
  - 実装量: 小（row.js / preview.js / quote-pdf.js / quote.css / index.html の5箇所）

- [ ] **見積テーブルのスプリットビュー対応（画面が狭い問題）**
  - 背景: スプリットビューで作業することが多く、見積テーブルの列が窮屈になる
  - 候補:
    - 列の表示/非表示トグル（メモ・CD等を折り畳める）← 費用対効果が高い
    - コンパクトモード（フォント・パディング縮小のCSSクラス切替）
    - 列幅ドラッグリサイズ
  - 優先候補: 列の表示/非表示トグル＋設定をlocalStorageに保存

## 🟡 低：将来アイデア・検討中

- [ ] **Supabase連携：サブコン別モジュール挿入**
  - 背景: 過去見積のサブコン別明細を流用・一部流用したい。保存の手間は増やしたくない
  - 設計方針（合意済み）:
    - 「モジュール保存」ステップは**作らない**。案件保存時に自動蓄積される設計
    - 行ごとに「サブコン名」列を追加するだけ（入力は任意でよい）
    - 挿入時: Supabase の全案件からサブコン名でグループ化 → 一覧表示 → 選んで挿入
    - 直近案件の行データが自動的に「最新モジュール」になる
  - 前提: Supabase への見積保存機能が先に必要
  - UIイメージ:
    ```
    「モジュール挿入」ボタン → パネルが開く
    - ABC Trucking（最終使用: 2026-05-30, 5行）
    - 横浜税関代行（最終使用: 2026-06-01, 3行）
    → 選ぶと現在のテーブルに行を挿入
    ```
