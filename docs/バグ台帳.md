# バグ台帳 — フォワーダー支援

最終更新: 2026-05-30 / 初版（3観点の棚卸しを統合）

このファイルは「使いながら開発するうちに、計算ミス・表示不備の修正に振り回される」状態を
脱するための地図。3つの観点（①計算ロジックの重複と不一致 ②計算の業界的正確性・データ品質
③表示・UI不整合）で棚卸しした結果を、重複を束ねて1枚にまとめたもの。

修正が完了した項目は `[x]` に変え、対応コミット/テストへのリンクを添える。

## 方針（このファイルの使い方）

- 高〜中の症状の大半は「**計算ロジックの散在**」と「**フォーマッタ/用語の複数系統**」の2つに収束する。
- つまり CLAUDE.md に持ち越されている「`shared/` への一本化」が、台帳の半分を一掃する作業を兼ねる。
- 進め方:
  - **Step 0** 軽量テスト基盤（ビルド不要・ブラウザで `フォワーダー支援/test/index.html` を開くだけ）
  - **Step 1** 計算を `shared/calc.js` に一本化 → A・F・G・H・I・J が「テストで守られた状態で」直る
  - **Step 2** フォーマッタ/用語を1系統へ集約 → B・E・K
  - **Step 3** 残りの状態・表示バグ（C・D・L・M・N）を個別に、テスト or 手動確認付きで
  - **Step 4** 知識系（O・P）はデータ構造見直しを伴うので別途相談

## 🔴 高：金額が静かに間違う / 客先に誤データが出る

- [x] **A. 未取得通貨が `rate=1` で暗黙換算され金額が桁外れ** ✅ 2026-05-30
  - 現象: `_fxRates[cur]` 未定義時に `|| 1` で「1外貨=1円」換算。例: 100万 KRW が ¥100万（約100倍ずれ）。利益・粗利率まで連鎖破壊。
  - 該当: `js/quote/constants.js:217`（`const rate = _fxRates[cur] || 1`）
  - あるべき: rate が無ければ換算せず「換算不可（レート未取得）」を明示。silent な rate=1 を廃止。
  - **対応**: 純粋関数 `SharedFX.toJPY(amount, cur, rates)` を新設し未取得時は `NaN` を返す（console.warn 付き）。`constants.js` の `toJPY` はそこへ委譲。NaN が合計に伝播 → 画面は「—」表示になり桁ずれを可視化。テスト `test/fx.spec.js`（9件）。

- [x] **B. 「粗利率」の定義が3画面で食い違い・ラベルが業界定義と逆** ✅ 2026-05-30
  - 現象: サマリ=原価ベース（profit/cost、例50%）、プレビュー警告/比較=売上ベース（profit/bill、例33%）。同案件で2つの率が出る。業界標準の「粗利率」は売上ベースなので、サマリ表示は実は「マークアップ率／値入率」でラベル誤り。
  - 該当: `js/quote/ui.js:1805` / `js/quote/preview.js:332,1389`
  - あるべき: 売上ベース=「粗利率」、原価ベース=「マークアップ率」と明記し分け、同一指標は同一式に統一。
  - **対応**: 共通関数 `SharedCalc.grossMarginPct(bill,cost)`（売上ベース）/ `markupPct`（原価ベース）を新設。3画面（ui.js サマリ・preview.js 警告・比較）を全て `grossMarginPct` に統一（ユーザー決定: 売上ベース）。テスト `test/margin.spec.js`（9件）。

- [x] **C. 御見積書PDFの数量が未入力でも「1」と印字される** ✅ 2026-05-30
  - 現象: `fmtNum(r.bq || r.pq || 1, 4)` で数量未入力時に強制「1」。客先提出物に誤データ。
  - 該当: `js/quote/quote-pdf.js:197`
  - あるべき: 未入力は空欄か「—」。勝手に1を補完しない。
  - **対応**: `qtyDisp = (r.bq && r.bq > 0) ? fmtNum(r.bq,4) : '—'` に変更。金額の根拠 `sub=bq×bp` と数量表示を一致させ「1」の捏造を廃止。同種の数量捏造は quote-pdf.js のみで、preview 等は既に `bq||0` で整合。

- [x] **D. FXフォールバック値の「確認日」が出力に出ない／中値で利益が甘く出る** ✅ 2026-05-30（一部・下記）
  - 現象: 取得失敗時はハードコードのデフォルトレートで換算するが、`getFxAuditMeta` は「手動設定値」とだけ出し、デフォルト値の確認日（コード上 2026-05-27）を出力しない。社外資料の根拠が追えない。加えて出典が中値（Mid）で、実決済 TTS/TTB と2〜3円乖離 → 原価が甘く出る。
  - 該当: `js/quote/constants.js:162`（DEFAULT_FX_RATES） / `js/quote/preview.js:131`（getFxAuditMeta） / `shared/fx.js:27`
  - あるべき: フォールバック時は「デフォルト値（最終更新 YYYY-MM-DD）」を明記。N日以上古ければ出力ブロック。中値である旨と TTS/TTB 乖離の注記。
  - **対応**: `DEFAULT_FX_RATES_ASOF`（確認日）を定数化し QuoteApp.fx 経由で公開。`getFxAuditMeta` のフォールバック文言に「基準レート確認日 YYYY-MM-DD」と「参考値・TTS等と異なる」注記を追加 → PDF/プレビュー/TSV/Excel の全出力に刻まれる。
  - **持ち越し（要・業務判断）**: 「N日以上古ければ出力ブロック」は挙動変更のため別途相談。中値→TTS/TTB のスプレッド加算オプションも未対応（台帳 D 残件）。

- [x] **E. 金額の桁区切りが画面ごとに混在（1,500,000 と 1500000）** ✅ 2026-05-30
  - 現象: テーブル本体・サマリは `fmt`（カンマ付き）、プレビュー本体/CSV/TSV は `fmtRaw`（カンマなし）。さらにプレビュー底部の税額・税込だけ `fmt` で、同一画面内に混在。
  - 該当: `js/quote/preview.js:652-653` / `js/quote/ui.js:142-145`（fmtRaw） / `js/quote/constants.js:143`（fmt）
  - あるべき: 金額表示は桁区切りを全体統一。
  - **対応**（ユーザー決定: 金額にカンマ・数量はカンマなし・CSV/TSVは据置）: 画面表示用 `fmtMoney`（3桁カンマ）を ui.js に新設。プレビューの金額セル（単価・CD・乗せ幅・小計・利益・JPY換算・税額・通貨別合計・総合計・底部サマリ）を `fmtMoney` に統一。数量セル（pq/bq）は `fmtRaw` のまま。TSV/CSV エクスポート定義は `fmtRaw` 維持（カンマが区切りと衝突するため）。御見積書PDFは既存 `fmtInt` で既にカンマ付き。
